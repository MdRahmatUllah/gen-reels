# Phase 7 Overview: Local And BYO Expansion

## Objective

Add advanced execution options so power users can route work through their own provider keys or registered local workers without changing the product workflow.

## Why This Phase Exists

This phase lowers costs for advanced users and broadens platform flexibility, but it should arrive only after the hosted flow is reliable. Otherwise the product risks becoming an operations-heavy infrastructure layer too early.

## In Scope

- BYO provider credentials
- Local worker registration and heartbeats
- Workspace execution policies
- Routing rules by modality
- Cost and security controls for hybrid execution

## Out Of Scope

- Full desktop app packaging
- Unbounded model marketplace behavior
- Self-hosted on-prem product packaging

## What Users Get

- The option to reduce hosted provider cost
- More control over execution strategy
- The same project workflow regardless of where generation runs

## Deliverables

- Encrypted BYO credential management
- Local worker agent contract
- Worker registration and health model
- Workspace routing policy controls
- Usage tracking across hosted and BYO paths

## Exit Criteria

- A workspace can choose hosted, BYO, or local execution for allowed modalities.
- Local workers can process jobs through the same orchestration model used by hosted providers.
- Credential handling remains auditable and secure.
